# edge cases

es importante tener en cuenta que no siempre todo saldrá como esperamos/planeamos

- happy path: es el camino quie recorre el usuarie perfecte. Con el final "feliz"

pero esto no será así para todes les usuaries.

los casos límite son situaciones o probelmas inesperaods que interrumpen la experiencia de usuario. muchas veces este obstáculo está fuera del control del usuarie.

## tips para  edge cases

- create personas and user sotires
- revisar el proyecto desde la perspectiva del usuarie
- use wireframes
